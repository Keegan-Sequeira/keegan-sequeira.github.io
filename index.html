Overview

This application is a remake of the classic game Tetris. It is coded in Python, and uses the PyGame framework. All Assets used in the game are under a free to use lisence and does not ask to be credited.

Installation

<em>1. Ensure you have the following installed (or newer):</em>

  • Python 3.11
  • Pygame 1.96

<em>2. Clone the repository:</em>

  • https://github.com/Keegan-Sequeira/3815ICT-Tetris.git

<em>3. Run the game:</em>

  • Open 'main.py' file OR
  • 'python main.py' within directory

To run the game, ensure the PyGame module is installed

Source Code Files

All the source code follow the camel case or pascal case naming convention depending on the data type. Where variable names and function names are camel case (e.g. userName), class names are pascal case (e.g. TetrominoPiece). Some of the constants are also defined and differentiated from by using full capitilisation (e.g. WINDOW_HEIGHT).

  • main.py (32 lines)

    This file runs the pyGame module and initiates the pyGame window and session.

  • main_menu.py (148 lines)

    The main menu python file defines the pyGame GUI and handles the user input events for the main menu. From the main menu, the user can navigate to the configure/settings page, the high scores page, the game page, and can also quit the application.

  • high_scores_menu.py (50 lines)

    This python file creates the GUI that displays the top 10 scores reached in the game. It can be navigated to from the main menu screen and clicking [ENTER] on that page will take the user back to the main menu.

  • settings_menu.py (116 lines)

    On this page, the user can edit the settings for the game. The settings set by the user remain persistent using a JSON file. The user can change the game size, game mode, game level, turn on the AI, and can mute the music.

  • gamePage.py (269 lines)

    This file displays the main game page, which includes the board and all other elements of the HUD. This script also handles the user input and communicates with the backend of the game.

  • backend.py (292 lines)

    The backend python module is used to create the board, create the tetromino pieces, move the pieces, rotate the pieces, check the move or rotation is allowed, checks if a line is full, and imports and exports JSON data.

  • colours.py (11 lines)

    This file contains some colour data that is used to colour in the Tetris pieces.

  • shapes.py (208 lines)

    This file defines the shape of each Tetromino, it also contains the class definition of the Tetromino class which has the following properties: x, y, shape, returnShape, rotaton, and number.

  • configure.json (1 line)

    Used to set game settings and keep user's selections persistent.
